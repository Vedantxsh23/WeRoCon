Step 12 — Read IMU Data

Create a Python script:

nano read_mpu.py

Paste this code:

import smbus2
import time

MPU6250_ADDR = 0x68
PWR_MGMT_1   = 0x6B
ACCEL_XOUT_H = 0x3B
GYRO_XOUT_H  = 0x43
TEMP_OUT_H   = 0x41

bus = smbus2.SMBus(1)
bus.write_byte_data(MPU6250_ADDR, PWR_MGMT_1, 0x00)
time.sleep(0.1)

def read_word(reg):
    high  = bus.read_byte_data(MPU6250_ADDR, reg)
    low   = bus.read_byte_data(MPU6250_ADDR, reg + 1)
    value = (high << 8) | low
    if value >= 0x8000:
        value -= 65536
    return value

def get_accel():
    ax = read_word(ACCEL_XOUT_H)     / 16384.0
    ay = read_word(ACCEL_XOUT_H + 2) / 16384.0
    az = read_word(ACCEL_XOUT_H + 4) / 16384.0
    return ax, ay, az

def get_gyro():
    gx = read_word(GYRO_XOUT_H)     / 131.0
    gy = read_word(GYRO_XOUT_H + 2) / 131.0
    gz = read_word(GYRO_XOUT_H + 4) / 131.0
    return gx, gy, gz

def get_temp():
    raw = read_word(TEMP_OUT_H)
    return (raw / 340.0) + 36.53

print("MPU6250 Live Data — Press Ctrl+C to stop\n")

try:
    while True:
        ax, ay, az = get_accel()
        gx, gy, gz = get_gyro()
        temp       = get_temp()
        print(f"Accel (g)    X:{ax:+.3f}  Y:{ay:+.3f}  Z:{az:+.3f}")
        print(f"Gyro (°/s)   X:{gx:+.3f}  Y:{gy:+.3f}  Z:{gz:+.3f}")
        print(f"Temp (°C)    {temp:.2f}")
        print("-" * 45)
        time.sleep(0.5)
except KeyboardInterrupt:
    print("Stopped.")

Save with Ctrl+X → Y → Enter, then run it:

python3 read_mpu.py

Gyro (°/s)   X:+0.213  Y:-0.152  Z:+0.031

CH 02

Angle Calculation, Low Pass Filter, Live Plotting & Calibration

Continuing from Chapter 01 — MPU6250 already connected and giving readings

✅
Prerequisites from Chapter 01 MPU6250 is connected and delivering accelerometer data (X, Y, Z) in m/s² and gyroscope data (X, Y, Z) in °/s.

1. Why We Need a Low Pass Filter

When you look at the accelerometer plot, even when the sensor is completely still, the readings are not a clean flat line — they shake around the true value.

Without Low Pass Filter:           With Low Pass Filter:

 Accel Z                            Accel Z
  |  ~~~~~~~~~~~~~~~~~~~~            |  ──────────────────
  |~~~~                              |
  └──────────────► time              └──────────────► time
  Noisy, hard to use                 Smooth, clean signal

This noise comes from vibration from motors or mechanical movement, electrical noise in the circuit, and sensor sensitivity picking up tiny movements. If we calculate angles from noisy data, the angles will also jump around wildly. The Low Pass Filter smooths out the noise by only allowing slow changes through and blocking fast spikes.

2. How the Low Pass Filter Works

Think of it like this: "The new reading should be mostly the old value, with just a little bit of the new raw reading mixed in."

Low Pass Filter Formula
filtered = alpha × raw_new + (1 - alpha) × filtered_old

Where alpha is a value between 0 and 1:

Alpha ValueEffect
alpha = 1.0No filtering — raw noisy signal passes through
alpha = 0.5Medium filtering — balanced
alpha = 0.1Heavy filtering — very smooth but slow to respond
alpha = 0.2Good for IMU — smooth but still responsive ✅

Example with numbers

Suppose the sensor is at rest (true value = 9.81 m/s²), but noise spikes to 11.5:

Without filter:  9.81 → 9.81 → 11.5 → 9.81   ← big jump visible
With filter:     9.81 → 9.81 → 9.93 → 9.87   ← spike absorbed smoothly

We use alpha = 0.2 — 80% weight on the previous smooth value (stability) and 20% weight on the new raw reading (responsiveness).

4. Full Code with Explanation

Create the main script:

nano live_angles.py

Paste this full code:

import smbus2, time, math, threading
from flask import Flask, jsonify, render_template_string

# ── SECTION 1: MPU6250 REGISTER ADDRESSES ──────────────────
MPU6250_ADDR = 0x68   # Default I2C address (AD0=GND)
PWR_MGMT_1   = 0x6B   # Power management register
ACCEL_XOUT_H = 0x3B   # Accel X high byte register

# ── SECTION 2: I2C BUS INITIALIZATION ──────────────────────
bus = smbus2.SMBus(1)
bus.write_byte_data(MPU6250_ADDR, PWR_MGMT_1, 0x00)  # Wake up
time.sleep(0.1)

# ── SECTION 3: READ RAW 16-BIT VALUE ───────────────────────
def read_word(reg):
    high  = bus.read_byte_data(MPU6250_ADDR, reg)
    low   = bus.read_byte_data(MPU6250_ADDR, reg + 1)
    value = (high << 8) | low
    if value >= 0x8000:
        value -= 65536
    return value

# ── SECTION 4: LOW PASS FILTER SETUP ───────────────────────
ALPHA = 0.2
ax_filtered = ay_filtered = az_filtered = 0.0

# ── SECTION 5: GET ACCELEROMETER DATA (with LPF) ───────────
def get_accel():
    global ax_filtered, ay_filtered, az_filtered
    ax_raw = (read_word(ACCEL_XOUT_H)     / 16384.0) * 9.81
    ay_raw = (read_word(ACCEL_XOUT_H + 2) / 16384.0) * 9.81
    az_raw = (read_word(ACCEL_XOUT_H + 4) / 16384.0) * 9.81
    ax_filtered = ALPHA * ax_raw + (1 - ALPHA) * ax_filtered
    ay_filtered = ALPHA * ay_raw + (1 - ALPHA) * ay_filtered
    az_filtered = ALPHA * az_raw + (1 - ALPHA) * az_filtered
    return ax_filtered, ay_filtered, az_filtered

# ── SECTION 6: ANGLE CALCULATION ───────────────────────────
# Roll  = atan2(ay, az)  — tilt left/right
# Pitch = atan2(-ax, sqrt(ay²+az²))  — tilt forward/back
timestamps = []; roll_data = []; pitch_data = []
start_time = time.time()

def calculate_angles():
    while True:
        ax, ay, az = get_accel()
        roll  = math.degrees(math.atan2(ay, az))
        pitch = math.degrees(math.atan2(-ax, math.sqrt(ay**2 + az**2)))
        elapsed = round(time.time() - start_time, 2)
        timestamps.append(elapsed)
        roll_data.append(round(roll, 3))
        pitch_data.append(round(pitch, 3))
        if len(timestamps) > 200:
            timestamps.pop(0); roll_data.pop(0); pitch_data.pop(0)
        time.sleep(0.05)

# ── SECTION 7: FLASK WEB SERVER + LIVE CHART ───────────────
app = Flask(__name__)
HTML_PAGE = """... (see full code in file) ..."""

@app.route('/')
def index(): return render_template_string(HTML_PAGE)

@app.route('/data')
def data(): return jsonify({'time': timestamps, 'roll': roll_data, 'pitch': pitch_data})

# ── SECTION 8: MAIN ─────────────────────────────────────────
if __name__ == '__main__':
    thread = threading.Thread(target=calculate_angles, daemon=True)
    thread.start()
    print("Low Pass Filter active! (alpha = 0.2)")
    print("Open your browser → http://raspberrypi.local:5000")
    app.run(host='0.0.0.0', port=5000)

Code Summary — What Each Section Does

SectionWhat it does
Section 1Defines I2C address and register locations inside the MPU6250 chip
Section 2Opens I2C bus and wakes up the sensor from sleep mode
Section 3Reads raw 16-bit integer by combining two 8-bit bytes
Section 4Sets up Low Pass Filter constant (alpha=0.2) and previous filtered values
Section 5Reads accelerometer, converts to m/s², applies Low Pass Filter
Section 6Calculates Roll & Pitch from filtered accelerometer using atan2
Section 7Flask web server serves live chart page and JSON data endpoint
Section 8Starts IMU thread and web server together

Filter Flow Diagram

Raw Accelerometer Reading (noisy) │ ▼ ┌───────────────────────┐ │ Low Pass Filter │ ← filtered = 0.2 × raw + 0.8 × prev │ alpha = 0.2 │ ← removes vibration and noise spikes └───────────────────────┘ │ ▼ ┌───────────────────────┐ │ atan2 Angle Formula │ ← Roll = atan2(ay, az) │ (from accel only) │ ← Pitch = atan2(-ax, sqrt(ay²+az²)) └───────────────────────┘ │ ▼ Clean Angle (°) ✅

6. Calibration — Why It Is Needed

Even with the Low Pass Filter applied, if the sensor has a built-in bias error, the angles will still be offset from the true value.

⚠️
Problem — Sensor Bias at Rest Even when completely still, the sensor shows: Accel X ≈ 63.8 m/s² (should be 0), Accel Y ≈ −2.9 m/s² (should be 0), Accel Z ≈ 11.8 m/s² (should be 9.81). This is manufacturing bias — the LPF smooths noise but cannot fix it.
Without CalibrationWith Calibration
Accel X at rest~63.8 m/s² ❌~0.0 m/s² ✅
Accel Y at rest~-2.9 m/s² ❌~0.0 m/s² ✅
Accel Z at rest~11.8 m/s² ❌~9.81 m/s² ✅
Angles accurateOffset ❌Correct ✅

Step 7.2 — Run Calibration Script

import smbus2, time

MPU6250_ADDR = 0x68
PWR_MGMT_1   = 0x6B
ACCEL_XOUT_H = 0x3B

bus = smbus2.SMBus(1)
bus.write_byte_data(MPU6250_ADDR, PWR_MGMT_1, 0x00)
time.sleep(0.1)

def read_word(reg):
    high  = bus.read_byte_data(MPU6250_ADDR, reg)
    low   = bus.read_byte_data(MPU6250_ADDR, reg + 1)
    value = (high << 8) | low
    if value >= 0x8000: value -= 65536
    return value

def get_accel():
    ax = (read_word(ACCEL_XOUT_H)     / 16384.0) * 9.81
    ay = (read_word(ACCEL_XOUT_H + 2) / 16384.0) * 9.81
    az = (read_word(ACCEL_XOUT_H + 4) / 16384.0) * 9.81
    return ax, ay, az

SAMPLES = 500
print(f"Collecting {SAMPLES} samples... Keep sensor FLAT and STILL!\n")
time.sleep(2)

ax_sum = ay_sum = az_sum = 0.0
for i in range(SAMPLES):
    ax, ay, az = get_accel()
    ax_sum += ax; ay_sum += ay; az_sum += az
    time.sleep(0.005)

ax_off = ax_sum / SAMPLES
ay_off = ay_sum / SAMPLES
az_off = (az_sum / SAMPLES) - 9.81  # subtract expected gravity

print("=" * 40)
print("CALIBRATION OFFSETS (copy these!):")
print("=" * 40)
print(f"ACCEL_X_OFFSET = {ax_off:.4f}")
print(f"ACCEL_Y_OFFSET = {ay_off:.4f}")
print(f"ACCEL_Z_OFFSET = {az_off:.4f}")
print("=" * 40)

CH 03

Two-State Kalman Filter Implementation

Orientation estimation with simultaneous gyro bias correction for the MPU9250/MPU6050

Introduction

This chapter implements a two-state linear Kalman Filter for orientation estimation using IMU data. The filter estimates both the angle and gyroscope bias simultaneously, making it significantly more accurate than simple gyro integration or a single-state Kalman Filter.

The implementation covers: State Space Models, State Prediction, Covariance Propagation, Process Noise Modeling, Measurement Updates, Kalman Gain Computation, and Sensor Fusion.

Why a Two-State Kalman Filter?

A gyroscope provides smooth angular velocity measurements but suffers from drift due to bias.

Example
Actual angular velocity0 deg/s
Gyroscope output0.5 deg/s ← constant offset = gyro bias

If gyro measurements are integrated without compensation, orientation estimates continuously drift over time. The Kalman Filter solves this by estimating Orientation Angle and Gyroscope Bias simultaneously.

State Vector

The state vector is defined as:

State Vector x
x = [ θ ] where θ = estimated angle [ b ] b = estimated gyroscope bias 
# Implementation
self.x = np.zeros((2,1))
# x[0] = angle,  x[1] = gyro_bias

State Transition Model

The angle evolves according to:

State Equation
θ_k = θ_(k-1) + (ω - b) × Δt where: ω = measured angular velocity b = estimated gyro bias

The state transition matrix becomes:

Transition Matrix F
F = [ 1 -dt ] [ 0 1 ] 
F = np.array([
    [1.0, -dt],
    [0.0, 1.0]
])

Control Matrix

Gyroscope measurements act as system input.

Control Matrix B
B = [ dt ] [ 0 ] 
B = np.array([
    [dt],
    [0.0]
])

Prediction Step

The Kalman Filter first predicts the next state before receiving a measurement.

Prediction Equation
x_k⁻ = F × x_(k-1) + B × u 
self.x = F @ self.x + B * gyro_rate

This produces a predicted estimate of angle and gyroscope bias.

Covariance Matrix

The covariance matrix represents uncertainty in the estimated states.

Covariance Matrix P
P = [ P_θθ P_θb ] [ P_bθ P_bb ] 
# Initial: moderate uncertainty
self.P = np.eye(2)
# P = [[1, 0], [0, 1]]

Process Noise Matrix

The process noise matrix models uncertainty in the prediction model.

Process Noise Q
Q = [ Q_angle 0 ] [ 0 Q_bias ] 
Q = np.diag([
    self.Q_angle,   # uncertainty in angle prediction
    self.Q_bias     # uncertainty in bias estimation
])

Covariance Propagation

After prediction, uncertainty increases because every prediction introduces possible error.

Covariance Propagation
P_k⁻ = F × P_(k-1) × F^T + Q 
self.P = F @ self.P @ F.T + Q

Measurement Model

The accelerometer provides an angle estimate. It directly measures angle but does not measure gyro bias.

Measurement
z = θ_acc H = [ 1 0 ] 
self.H = np.array([
    [1.0, 0.0]
])

Residual Calculation

The residual represents the disagreement between prediction and measurement.

Residual
y = z - H × x If prediction and measurement agree → Residual ≈ 0 If they disagree → Residual becomes large 
y = meas_angle - (H @ self.x)

Dynamic Measurement Noise

Instead of a fixed measurement noise covariance R, this implementation uses R_dynamic.

update(gyro_rate, meas_angle, R_dynamic)
ConditionBehaviour
Leg is stationaryAccelerometer is trusted more → Small R
Leg is moving rapidlyAccelerometer becomes noisy → Large R

This improves robustness during gait analysis.

Innovation Covariance

Represents expected uncertainty in the residual.

Innovation Covariance
S = H × P × H^T + R 
S = (H @ P @ H.T) + R_dynamic

Kalman Gain

The Kalman Gain determines how much correction should be applied.

Kalman Gain
K = P × H^T × S⁻¹ 
self.K = (P @ H.T) / S
Kalman Gain KMeaning
Large KTrust measurement more
Small KTrust prediction more

State Update

The state estimate is corrected using the residual. Both angle and gyro bias are corrected simultaneously.

State Update
x⁺ = x⁻ + K × y 
self.x = self.x + self.K * y

Covariance Update

After incorporating a measurement, uncertainty decreases.

Covariance Update
P⁺ = (I - K × H) × P⁻ 
self.P = (np.eye(2) - self.K @ self.H) @ self.P

Estimated Outputs

Angle Estimate θ

self.x[0, 0]

Gyroscope Bias Estimate b

self.x[1, 0]

# Accessed via property:
@property
def bias(self):
    return float(self.x[1,0])

Conclusion

A two-state Kalman Filter was implemented where the state vector consists of orientation angle and gyroscope bias. Gyroscope measurements are used during prediction, while accelerometer-derived angles are used during measurement updates. Dynamic measurement noise adaptation improves performance during periods of high motion, making the filter suitable for real-time human gait analysis and wearable sensing applications.
