MODULE 02 // ACTUATION

Motor Control

Three subsystems work in sequence to drive the ankle actuator: the Raspberry Pi is set up as a headless compute node, a CAN bus bridges it to the motor, and the CubeMars AK80-9 is configured and tuned via Servo and MIT control modes.

CH 01 Raspberry Pi Setup CH 02 CAN Bus Setup CH 03 CubeMars AK80-9
Chapter 01 — Compute Node

Raspberry Pi Setup

The prosthetic leg runs headless — no monitor, keyboard, or mouse. This chapter covers OS flashing, first-boot wireless configuration, and establishing SSH and VNC remote access for development.

Hardware Requirements
ComponentSpecification
Raspberry PiRaspberry Pi 4 or Raspberry Pi 5
MicroSD Card32 GB or larger (Class 10 / A1 rated)
SD Card AdapterUSB or full-size SD adapter for host computer
Host ComputerWindows / Linux / macOS on the same Wi-Fi network
Power SupplyOfficial Raspberry Pi adapter or regulated battery source
Flashing the OS
STEP 01

Flash Raspberry Pi OS

Use Raspberry Pi Imager v2.0.7 to write the OS image to the MicroSD card.
  1. Insert the MicroSD card into the host computer via an SD adapter.
  2. Open Raspberry Pi Imager and allow administrator permissions if prompted.
  3. Select: Device → your Pi model · OS → Raspberry Pi OS (64-bit) · Storage → your MicroSD card.
  4. In the advanced settings (gear icon), configure hostname, username, password, Wi-Fi credentials, and enable SSH before writing.
  5. Click Write and wait for verification to complete.
  6. Safely eject the MicroSD card.
STEP 02

First Boot

Insert the MicroSD card into the Raspberry Pi, connect power, and wait 2–3 minutes for the system to initialise and connect to Wi-Fi.
Remote Access — SSH
STEP 03

Connect via SSH (Windows — PuTTY)

  1. Open PuTTY.
  2. Set Hostname to prosthetic-leg.local, Port to 22, Connection Type to SSH.
  3. Click Open and log in with your configured username and password.

On Linux / macOS, open a terminal and run:

ssh <username>@prosthetic-leg.local
Remote Access — VNC (Graphical)
STEP 04

Enable VNC

Run over SSH:
sudo raspi-config nonint do_vnc 0
If you are on Raspberry Pi OS Bookworm and VNC shows a black screen, switch to X11:
sudo raspi-config nonint do_wayland W1 sudo reboot
Then open RealVNC Viewer on your host computer and connect to:
prosthetic-leg.local
ToolPurpose
SSH (PuTTY)Command-line access — primary development interface
VNC ViewerRemote desktop access when a GUI is needed

Chapter 02 — Communication Bus

CAN Bus Setup

The CubeMars AK80-9 communicates over CAN at 1 Mbps. This chapter covers physical wiring of the CAN HAT, enabling the MCP2515 driver, automating the interface on boot, and verifying the link with a loopback test before connecting the motor.

Hardware Connections
STEP 01

Wire the CAN HAT to the Motor

Mount a CAN HAT (e.g. Waveshare RS485/CAN HAT) on the Raspberry Pi GPIO header, then connect the motor's XT30 2+2 connector:
  • CAN_H on HAT → CAN_H (White wire) on motor
  • CAN_L on HAT → CAN_L (Blue wire) on motor
  • Battery power → motor via XT60-to-XT30 power lines
Driver & Library Configuration
STEP 02

Enable the MCP2515 SPI Driver

Open the boot configuration file over SSH:

Raspberry Pi OS Bookworm:

sudo nano /boot/firmware/config.txt

Older OS (Bullseye or earlier):

sudo nano /boot/config.txt
Scroll to the bottom and add:
dtparam=spi=on dtoverlay=mcp2515-can0,oscillator=12000000,interrupt=25,spimaxfrequency=2000000
Note on oscillator: Check the metal crystal on your CAN HAT. If it reads 12.000 use oscillator=12000000. If it reads 8.000 use oscillator=8000000.
Save (Ctrl+O, Enter), exit (Ctrl+X), then reboot:
sudo reboot
STEP 03

Automate CAN Interface on Boot

Create a network interface config so can0 starts at 1 Mbps on every boot:
sudo nano /etc/network/interfaces.d/can0
Paste:
auto can0 iface can0 can static bitrate 1000000 up /sbin/ip link set $IFACE txqueuelen 1000
Save and exit.
STEP 04

Install Utilities & Python Library

# Command-line CAN tools sudo apt update && sudo apt install can-utils -y # Python SocketCAN library sudo apt install python3-can -y
STEP 05

Set the Motor's CAN ID

Before connecting to the Pi, assign the motor its CAN address via the R-Link:
  1. Connect the motor to your PC using the R-Link module.
  2. Open CubeMars AK Config Tool.
  3. Under Application Configuration, set CAN ID (e.g. 104 / 0x68).
  4. Click Write to save to the motor's flash memory.
Loopback Validation (Without Motor)
STEP 06

Loopback Communication Check

Verify the Pi's CAN controller, transceiver, and libraries work before powering the motor.
# 1. Bring the interface down sudo ip link set can0 down # 2. Enable loopback mode sudo ip link set can0 type can bitrate 1000000 loopback on # 3. Bring it back up sudo ip link set can0 up
Open two SSH terminals:
  • Terminal 1 — listen:
    candump can0
  • Terminal 2 — send a test packet:
    cansend can0 123#DEADBEEF
Expected output in Terminal 1:
can0 123 [4] DE AD BE EF
If this appears, the CAN stack is fully functional.
STEP 07

Switch to Normal Mode (Connect Motor)

# Take interface down sudo ip link set can0 down # Disable loopback sudo ip link set can0 type can bitrate 1000000 loopback off # Bring back up — Pi is now live on the CAN bus sudo ip link set can0 up
Chapter 03 — Actuator

CubeMars AK80-9 V3.0

The AK80-9 is a high-torque BLDC actuator with an integrated controller, 9:1 planetary gearbox, and dual encoder feedback. This chapter covers initial hardware setup, driver board calibration, Servo Mode, and MIT Mode.

Technical Specifications
ParameterSpecification
Input Voltage48 V (range: 18–52 V)
Rated Output Current (RMS)20 A
Peak Output Current60 A
CAN Bus Bitrate1 Mbps
Inner Encoder21-bit Single-turn Absolute
Outer EncoderOptional 15-bit Single-turn Absolute
Communication InterfaceCAN + UART
Ambient Temperature Range-20 °C to 65 °C
Max Driver Temperature100 °C
Initial Hardware Setup
STEP 01

Physical Connections

  1. Connect the AK80-9 actuator to the driver board.
  2. Connect the R-Link module to the driver's communication port.
  3. Connect the R-Link USB interface to the host computer.
  4. Connect the power supply (18–52 V).
  5. Launch the CubeMars AK Config Tool.
STEP 02

Establish Communication

  1. Navigate to the Connection panel in the software.
  2. Select the appropriate COM port.
  3. Set baud rate to 921600.
  4. Click Connect.
  5. Verify that real-time motor parameters are actively updating in the dashboard.
Driver Board Calibration
When to calibrate: Recalibration is required only after reinstalling the driver board, changing the three-phase wiring sequence, or updating firmware. The motor is factory-calibrated and does not need this out of the box.
STEP 03

Calibration Procedure

  1. STEP 0 — Confirm stable power supply and connection; enter the System Settings page.
  2. STEP 1 — Click Read and wait for the connection interface.
  3. STEP 2 — Click Motor Identification. After a brief beep the motor rotates for ~10 seconds. When the status message appears, motor parameter identification is complete.
  4. STEP 3 — Click Encoder Identification. The motor rotates slowly for ~45 seconds. Wait for the completion message.
  5. STEP 4 — Click Write to save calibration to flash.
⚠ Warning: Both identification steps must be performed under no-load conditions. Running encoder identification multiple times consecutively can cause sharp temperature rise in the motor.
Servo Mode

Servo Mode offloads position and velocity loops to the actuator's internal driver. Useful for testing joint configurations and calibrating absolute angle limits.

Duty Cycle Mode

Controls average motor voltage through PWM.

V_avg = D × V_supply
Current Loop Mode

Regulates motor current to control torque.

T = Kt × I
Velocity Loop Mode

Controls motor speed via error feedback.

e_v = ω_ref − ω_actual
Position Loop Mode

Controls shaft angle.

e_p = θ_ref − θ_actual
Position-Velocity Loop

Position control with velocity and acceleration constraints.

|ω| ≤ ω_max, |α| ≤ α_max
Current Brake Mode

Generates braking torque via current regulation.

T_brake = Kt × I_brake
STEP 04

Running Servo Mode

  1. Open the Servo Control tab.
  2. Select the desired control mode.
  3. Configure the required command parameters (position, velocity, acceleration limits as applicable).
  4. Click Run to execute.
  5. Monitor motor response in the real-time status window.

Origin configuration: Move the actuator to the desired reference position and click Set Origin to store it as zero. Click Back Origin to return to it.

⚠ Warning: The motor may move at high speed when returning to origin. Ensure the workspace is clear before executing this command.
MIT Mode

MIT Mode combines position, velocity, and torque control in a single control law — the primary mode used during gait-phase-driven actuation.

T = Kp(θ_ref − θ_actual) + Kd(ω_ref − ω_actual) + T_ff
STEP 05

MIT Mode Configuration & Testing

  1. Navigate to the MIT Control tab and confirm communication is active.
  2. Set the required parameters: Position (P), Velocity (S), Feedforward Torque (T), KP, KD, Motor ID.
  3. Stiffness tuning (KP): Start low, increase gradually, observe position-holding stiffness.
  4. Damping tuning (KD): Hold KP constant, increase KD incrementally to reduce oscillation and overshoot.
  5. Velocity-only test: Set KP = 0, configure KD, enter desired velocity and execute.
  6. Torque-only test: Set KP = KD = 0, enter feedforward torque value and execute.

Real-time parameters to monitor during testing:

  • Position · Velocity · Torque · Phase Current
  • Bus Voltage · Motor Temperature · MOSFET Temperature
Emergency Stop: Press STOP immediately if abnormal behaviour is observed. Disconnect power if needed. Verify all parameter values before restarting.
Firmware Update
STEP 06

Update Firmware via AK Config Tool

  1. Select the corresponding firmware from the drop-down list.
  2. Click Jump to IAP.
  3. Click Upload and wait for the progress bar to reach 100%.
  4. Click Jump to App and wait 5 seconds for the motor to enter operational mode.
